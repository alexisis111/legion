# Инструкции для Qwen CLI

## 1. Язык общения
- Все ответы должны быть на русском языке.
- Если вопрос задан на другом языке, переводите его на русский и отвечайте на русском.

## 2. Подход к решению задач
- Используйте простой и последовательный подход к решению задач.
- Избегайте грубых или неопределенных предположений.
- Разбивайте решение на шаги, если это возможно.

## 3. Работа с файлами во внутреннем редакторе
- После внесения изменений в файл через внутренний редактор:
    - Показывайте diff-изменения для каждого файла.
    - Используйте формат `-` для удаленных строк и `+` для добавленных строк.
    - Пример показа diff:
      ```diff
      --- a/file.js
      +++ b/file.js
      @@ -1,5 +1,6 @@
       function calculateSum(a, b) {
      -  return a + b;
      +  const result = a + b;
      +  console.log(result);
         return result;
       }
      ```
    - Запрашивайте одобрение пользователя перед сохранением изменений.
        - Формат запроса: `"Были внесены изменения в файл file.js. Одобрить эти изменения? (y/n)"`.
        - Если пользователь выбирает `n`, отменяйте изменения.

## 4. Написание кода
- После написания кода всегда проводите проверку работоспособности.
- Включайте команды для запуска тестов или проверок в свой ответ.
- Убедитесь, что изменения не ломают существующую функциональность.

## 5. Форматирование ответов
- Используйте четкое и понятное форматирование.
- Для кода используйте блоки с указанием языка программирования.
- Добавляйте комментарии к сложным частям кода.

## 6. Обработка ошибок
- Если возникают ошибки, предоставляйте подробные объяснения их причины.
- Предлагайте конкретные шаги для исправления ошибок.

## 7. Дополнительные правила
- Если запрос неясен, уточняйте детали перед началом работы.
- Не делайте лишних предположений о задаче.
